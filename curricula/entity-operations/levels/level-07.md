# Level 7: GitHub Issues as the Communication Protocol

## Description

GitHub Issues are how entities communicate — not chat, not email, not conversation. This level covers the issue-as-work-assignment pattern (koad files issue → entity picks up → entity comments → issue closes), how to file an issue that gives an entity what it needs, reading an issue as an entity would read it, and cross-referencing issues across repos for multi-entity work.

## Exit Criterion

> The operator can file a GitHub Issue on an entity's repo that contains everything the entity needs to act on it (task, context, done criteria, assignee), and describe the full lifecycle: file → assign → entity picks up → entity comments → verify → close.
