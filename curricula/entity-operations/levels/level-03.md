# Level 3: Writing a Good Task Prompt

## Description

A task prompt is not a chat message — it is a brief. This level covers what information an entity needs to act, the difference between an orientation prompt and a task prompt, why context must be explicit (the entity does not remember the last conversation unless memory was committed), and the patterns that cause tasks to fail silently.

## Exit Criterion

> The operator can write a task prompt for a real entity (e.g., "research X and write a findings file") that includes: what to do, where to put the output, what done looks like, and any constraints — and can explain why each element is necessary.

---

## Atoms (stubs)

- **3.1** — What an entity knows at session start: CLAUDE.md, memories/, and nothing else from prior sessions
- **3.2** — The anatomy of a task prompt: task, output location, done criteria, constraints
- **3.3** — Orientation vs task: when to provide background vs when to just give the task
- **3.4** — Failure patterns: vague tasks, missing output location, no done criterion
- **3.5** — Scoping for entity specialty: matching the task to the entity and why this matters
