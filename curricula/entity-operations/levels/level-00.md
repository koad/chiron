# Level 0: Before the Session — Environment Check

## Description

Before spawning an entity, the operator must verify that their environment is ready: the entity directory exists and is current, the framework is reachable, and the correct identity is loaded. This level teaches the pre-flight sequence that prevents most new-operator errors.

## Exit Criterion

> The operator can walk through the pre-session checklist from memory — entity directory pulled, framework present, identity confirmed — and explain why each step matters before they would ever run an entity.

---

## Atoms (stubs)

- **0.1** — What "environment ready" means and why it matters before any session
- **0.2** — The entity directory: git pull before every session, no exceptions
- **0.3** — Verifying the framework layer is present and up to date
- **0.4** — Identity check: whoami, hostname, and why the wrong host is a stop condition
